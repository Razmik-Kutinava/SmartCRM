# Eval Hermes — лиды, задачи, прочее

Набор для проверки роутера `backend/core/hermes.py`: текст команды → JSON с `intent` и `slots`.

## Интенты

| Группа | Интент | Смысл |
|--------|--------|--------|
| Лиды | `create_lead` | Новый лид |
| Лиды | `update_lead` | Изменить поля лида |
| Лиды | `delete_lead` | Удалить лид |
| Лиды | `list_leads` | Список / фильтр лидов |
| Задачи | `create_task` | Новая задача |
| Задачи | `list_tasks` | Список задач |
| Задачи | `update_task` | Изменить задачу |
| Задачи | `delete_task` | Удалить задачу |
| Прочее | `write_email` | Письмо (маркетолог) |
| Прочее | `run_analysis` | Аналитика |
| Прочее | `ask_strategist` | Вопрос стратегу |
| Прочее | `search_web` | Поиск в интернете |
| Прочее | `noop` | Не CRM-команда |

## Слоты (эталон в кейсе)

В `cases.jsonl` для каждого примера заданы **ожидаемые** ключи в `expected_slots`. Это не «все возможные поля», а то, что должно быть извлечено из фразы для **честной оценки**. Пустые значения в эталоне не используются.

## Синонимы и «говорю как попало»

Роутер должен переносить смысл, а не дословный текст. В скрипте сравнения:

- **Телефоны** нормализуются до цифр (8→7 для 11-значных).
- **Текст** — приведение к нижнему регистру, обрезка пробелов.
- **Частичное совпадение**: если эталонная подстрока входит в ответ модели или наоборот — балл **0.5** за слот; полное совпадение после нормализации — **1.0**.

## Ошибки и частичный балл

| Проблема | Как отражается в скоре |
|----------|-------------------------|
| Неверный `intent` | `intent_ok = 0`, итог сильно падает (см. формулу в `run_eval.py`) |
| Верный intent, слот пропущен | балл по этому слоту 0 |
| Верный intent, слот частично | 0.5 за слот |
| Верный intent, все слоты | близко к 1.0 по кейсу |

Формула по умолчанию: **40%** совпадение интента + **60%** среднее по слотам. Если интент неверный, слоты всё равно считаются с коэффициентом **0.35** (модель могла вытащить полезный контекст).

## Запуск

Из корня репозитория (Windows или WSL), с активированным venv бэкенда и переменными из `.env`:

```bash
cd /path/to/SmartCRM
python eval/run_eval.py
```

Опции:

```text
--cases eval/cases.jsonl   путь к кейсам
--verbose                  печать ответов модели по каждому кейсу
```

Нужны доступные **Groq** или **Ollama** (как в `hermes.py`).

## Формат `cases.jsonl`

Одна строка — один JSON:

```json
{
  "id": "eval-001",
  "text": "фраза пользователя",
  "expected_intent": "create_lead",
  "expected_slots": {"company": "Ромашка", "phone": "+79161112233"},
  "tags": ["lead", "messy"]
}
```

Поле `tags` опционально (для отчёта).
