# Email Integration Test Plan

## Цели
- Проверить корректность синхронизации почты (Yandex, Gmail, др.)
- Проверить обработку ошибок при неудачной синхронизации
- Проверить отображение ошибок на фронте
- Проверить работу вкладок (Входящие, Отправленные, Кампании, Архив)
- Проверить массовую рассылку из профиля лида
- Проверить отсутствие дублирования аккаунтов

## Тест-кейсы

### 1. Подключение почтового аккаунта
- [ ] Подключить Yandex (валидные данные)
- [ ] Подключить Gmail (валидные данные)
- [ ] Подключить с неверным паролем (ожидается ошибка)
- [ ] Подключить уже существующий аккаунт (не должно быть дубля)

### 2. Синхронизация писем
- [ ] После подключения должны загрузиться письма (Inbox, Sent)
- [ ] При ошибке синхронизации — ошибка на фронте
- [ ] Проверить корректность отображения писем (отправитель, тема, дата)

### 3. Вкладки почты
- [ ] Переключение между Inbox, Sent, Campaigns, Archive
- [ ] Вкладка Campaigns отображает рассылки
- [ ] Вкладка Archive работает (архивировать/разархивировать письмо)

### 4. Массовая рассылка
- [ ] Открыть профиль лида, перейти в Campaign
- [ ] Отправить рассылку нескольким лидам
- [ ] Проверить доставку и отображение отправленных писем

### 5. Обработка ошибок
- [ ] Ошибка IMAP/SMTP отображается на фронте
- [ ] Ошибка при отправке кампании отображается

### 6. UI/UX
- [ ] Нет дублирующихся аккаунтов
- [ ] Корректная навигация между вкладками
- [ ] Сообщения об ошибках понятны пользователю

## Примечания
- Для тестов с реальной почтой использовать тестовые ящики
- Для массовой рассылки — тестовые email-адреса
- Все ошибки должны быть видны пользователю
