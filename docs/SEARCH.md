**Реализовано:** 10 апреля 2026  
**Статус:** Production-ready  
**Тесты:** 5/5, 19/19 проверок — 100%

## Архитектура

```
Запрос (компания / свободный текст / задача)
    ↓
backend/rag/search.py
    ├─ Serper   — Google-результаты, основной провайдер
    ├─ Brave    — независимый индекс, дополнительный
    └─ Tavily   — AI-поиск, читает страницы и суммаризирует
         ↓ (параллельно, asyncio.gather)
    Дедупликация (домен ≤2 + первые 80 символов сниппета)
         ↓
    Фильтр по дате (последние N месяцев, настраивается)
         ↓
    LLM-реранкинг → топ-K результатов
         ↓
    Форматированный блок для LLM-промпта
         ↓
backend/rag/retrieve.py → attach_rag_to_slots()
    → агент получает контекст в _rag_{agent}
```

## 6 режимов поиска

### 1. Поиск по компании (`/api/search/run`)
Ищет всё о конкретной компании. Каждый агент получает свои целевые запросы.

```json
POST /api/search/run
{ "company": "Сбербанк", "agent": "analyst", "industry": "финансы", "force": false }
```

**Возвращает:** `raw_results`, `formatted_block` (для LLM-промпта), `cached`, `providers_used`.

---

### 2. Свободный запрос (`/api/search/ask`)
Любой вопрос → поиск по всем провайдерам → LLM-ответ с источниками.

```json
POST /api/search/ask
{ "query": "Какие CRM чаще используют в ритейле России?", "summarize": true }
```

---

### 3. Проспектинг (`/api/search/prospect`)
По описанию ICP (идеального клиента) находит потенциальные компании + скорит их.

```json
POST /api/search/prospect
{
  "icp": "Торговые компании с оборотом от 100 млн, 20-200 сотрудников, без CRM",
  "industry": "ритейл", "city": "Москва", "count": 10
}
```

**Возвращает:** `companies` — список `{ name, snippet, url, fit_score, fit_reason }`.  
Кнопка «+ В CRM» добавляет компанию в базу лидов через `POST /api/leads`.

---

### 4. Обогащение лида (`/api/search/enrich-lead`)
Для компании из базы ищет: телефон, email, сайт, выручку, ЛПР, новости за неделю.

```json
POST /api/search/enrich-lead
{ "lead": { "company": "РетейлМаркет", "industry": "ритейл" } }
```

**Возвращает:** `enriched` — словарь `{ поле: значение }`.

---

### 5. Поиск для RAG (`/api/search/find-for-rag` + `POST /api/rag/ingest-batch`)
Ищет статьи, PDF, документацию → показывает превью → пользователь отмечает нужное → одна кнопка загружает в Chroma.

```json
POST /api/search/find-for-rag
{ "query": "книги по B2B продажам", "content_type": "pdf" }

// После апрува:
POST /api/rag/ingest-batch
{
  "documents": [{ "text": "...", "metadata": { "title": "...", "source": "..." } }],
  "for_agent": "analyst",
  "tags": "search"
}
```

`for_agent` — загружает в RAG конкретного агента или `all`.

---

### 6. Задача агенту через поиск (`/api/search/agent-task`)
ReAct-паттерн: агент сам генерирует 3-5 запросов → выполняет → синтезирует ответ.

```json
POST /api/search/agent-task
{
  "task": "Найди топ-5 конкурентов Сбербанка в B2B-кредитовании",
  "agent_id": "analyst",
  "context": "регион Москва"
}
```

**Возвращает:** `queries_used`, `answer`, `raw_results`.

## Провайдеры

| Провайдер | Лимит (бесплатно) | Роль | Env-переменная |
|---|---|---|---|
| **Serper** | 2 500 запросов/мес | Google-результаты, основной | `SERPER_API_KEY` |
| **Brave** | 2 000 запросов/мес (1 req/s) | Независимый индекс | `BRAVE_API_KEY` |
| **Tavily** | 1 000 запросов/мес | AI-поиск + суммаризация страниц | `TAVILY_API_KEY` |

Если провайдер недоступен (нет ключа / 429) — остальные продолжают работать.

## Умные запросы по агентам

| Агент | Шаблоны |
|---|---|
| **analyst** | `{company} CRM продажи автоматизация`, `{company} выручка финансы` |
| **economist** | `{company} бюджет инвестиции сделки`, `{company} финансовый отчет` |
| **marketer** | `{company} новости 2024 2025`, `{company} маркетинг конкуренты` |
| **tech_specialist** | `{company} IT стек технологии`, `{company} вакансии программист` |
| **default** | `{company} официальный сайт`, `{company} новости` |

Шаблоны редактируются в `/ops/search` без перезапуска бэкенда.

## Улучшения релевантности (все реализованы)

1. **Умные запросы** — каждый агент получает целевые запросы под свою задачу
2. **3 провайдера параллельно** — разные индексы, меньше шума
3. **Дедупликация** — по домену (≤2 с одного) + по первым 80 символам сниппета
4. **Фильтр даты** — убирает материалы старше N месяцев (настраивается, 0 = без фильтра)
5. **Структурированный формат** — не сырые HTML-сниппеты, а чистый блок фактов для LLM
6. **LLM-реранкинг** — после сбора 15-20 результатов модель выбирает топ-K наиболее релевантных
7. **24h кэш** — повторный запрос по той же компании мгновенный, API не тратятся
8. **Интеграция в RAG-контекст** — агенты видят веб-данные в промпте автоматически

## Настройка через /ops/search

Полный контроль без правки кода:

- **Провайдеры** — включить/отключить, настроить вес и макс. результатов
- **Реранкинг** — включить/выключить LLM, задать top-K
- **Фильтр даты** — количество месяцев назад (0 = без фильтра)
- **Кэш** — TTL в часах, просмотр и очистка
- **Шаблоны запросов** — для каждого агента, добавить/удалить/изменить
- **Тест-поиск** — проверить что возвращают провайдеры прямо в UI

## Запуск тестов

```bash
cd backend
python tests/test_search.py
```

**Тесты покрывают:**
- S1: Базовый поиск (Сбербанк, analyst) — результаты, форматирование, провайдеры
- S2: Кэш — повторный запрос возвращает из кэша
- S3: Другой агент (marketer) — другие запросы, нет дублей по домену
- S4: Дедупликация — нет дублей сниппетов и доменов (Газпром, economist)
- S5: Конфиг — структура, все провайдеры, шаблоны, реранкинг

## REST API: полный список endpoints

```
GET  /api/search/config          — текущий конфиг
PUT  /api/search/config          — сохранить конфиг
GET  /api/search/providers       — статус провайдеров (есть ли ключи)
GET  /api/search/cache           — список кэшированных записей
DELETE /api/search/cache         — очистить кэш

POST /api/search/run             — поиск по компании + агент
POST /api/search/ask             — свободный запрос + LLM-ответ
POST /api/search/prospect        — проспектинг по ICP
POST /api/search/enrich-lead     — обогащение лида
POST /api/search/find-for-rag    — поиск материалов для RAG
POST /api/search/agent-task      — задача агенту через ReAct-поиск

POST /api/rag/ingest-batch       — batch-загрузка документов из поиска в Chroma
```
