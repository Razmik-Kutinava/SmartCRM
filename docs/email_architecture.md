# Email-модуль SmartCRM: архитектура, задачи, тесты

## Основные задачи
- Надёжная интеграция с почтой (Яндекс, Gmail, Mail.ru)
- Вкладки: Входящие, Исходящие, Рассылки, Архив
- Массовая рассылка из профиля лида и кампании
- Фильтрация и хранение только нужных данных (архивация)
- Защита приватных данных (никаких лишних протоколов наружу)

## Проблемы, которые были решены
- Ошибка MailBox: неверный параметр ssl (теперь используется MailBoxSSL)
- Backend не сохраняет аккаунт, если не прошла синхронизация
- Ошибки синхронизации теперь видны пользователю
- Дубли аккаунтов в UI устранены
- Кампании отображаются и создаются из UI

## Архитектура UI
- `/email/[mailbox].svelte` — вкладки (Входящие, Исходящие, Рассылки, Архив)
- `/leads/[leadId]/campaign.svelte` — массовая рассылка по лиду
- Все данные подгружаются через API, фильтрация по категории

## Архитектура backend
- `/api/email/accounts/connect` — подключение и sync
- `/api/email/threads` — все треды, фильтрация по категории
- `/api/email/campaigns` — все кампании
- `/api/email/send` — отправка письма
- `/api/email/bind-lead` — привязка к лиду

## Тесты (минимум)
- Подключение Яндекс-почты с app-password
- Получение 1000 писем за 2 года (inbox, sent, campaign)
- Создание кампании из профиля лида
- Проверка фильтрации по категориям
- Проверка ошибок (неправильный пароль, IMAP)
