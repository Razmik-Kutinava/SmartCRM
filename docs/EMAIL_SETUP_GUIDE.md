## 📝 Как подключить Яндекс Почту

### Шаг 1: Создать app-password в Яндекс
1. Перейди на https://id.yandex.ru/security/app-passwords
2. Выбери приложение "Не указано" или создай новое
3. Скопируй сгенерированный пароль (16 символов)

### Шаг 2: Заполнить форму в SmartCRM
Перейди на `http://localhost:5173/email`

Форма "Подключить почту":
```
Название аккаунта:     Моя Яндекс Почта
Email:                 твой_email@yandex.com
Пароль / app password: [app-password из Яндекса]
IMAP сервер:           imap.yandex.com      (предзаполнено)
IMAP порт:             993                   (предзаполнено)
SMTP сервер:           smtp.yandex.com      (предзаполнено)
SMTP порт:             465                   (предзаполнено)
```

### Шаг 3: Нажми кнопку
Кнопка "Подключить и синхронизировать"

Система:
1. Подключится к твоей почте по IMAP
2. Загрузит письма
3. Привяжет письма к лидам (если есть совпадение email)
4. Покажет результат внизу экрана

## 🔗 Привязка email к лидам

После подключения почтового аккаунта:

1. Раздел "Привязка почты к лидам"
2. Выбери:
   - **Почтовый аккаунт** из выпадающего списка
   - **Лида** из выпадающего списка (все твои лиды)
3. Нажми "Привязать почту к лиду"

**Что это делает:**
- Система будет автоматически привязывать письма к этому лиду
- Письма от его email будут отнесены к его профилю

## 📧 Создание кампании рассылок

Раздел "Создать кампанию рассылок" с двумя колонками:

### Левая колонка (форма):
```
Аккаунт для отправки:  [выпадающий список]
Название кампании:     Летняя акция 2026
Тема письма:          Получай скидку 20% на все услуги
Текст письма:         [большое поле для текста письма]
```

### Правая колонка (выбор лидов):
- Checkboxes всех доступных лидов
- Показывает сколько лидов выбрано: "Выбранные лиды (5)"
- Можешь скролить список если много лидов

**Как использовать:**
1. Выбери аккаунт откуда отправлять
2. Заполни название, тему и текст письма
3. Отметь лидов которым нужно отправить письмо
4. Нажми "Создать кампанию"
   - `send_now: false` - письма создаются в статусе "draft"
   - Позже можешь отправить все сразу

## 🐛 Отладка (если что-то не работает)

### Открыть DevTools
Нажми **F12** в браузере → вкладка **Console**

Там ты увидишь:
```
Попытка подключения: {name, username, imap_server, ...}
Отправка запроса: {...payload...}
Результат подключения: {account: {...}, sync: {...}}
```

### Проверить backend логи
В терминале где запущен backend смотри ошибки:
```
ERROR: IMAP connection failed for user@yandex.com
WARNING: Email sync failed: ...
```

### Типичные ошибки

**❌ Ошибка: Invalid email in request**
- Проверь что в поле Email правильный формат: `user@yandex.com`
- Без пробелов в начале/конце

**❌ Ошибка: IMAP connection refused**
- Проверь что используешь app-password, не обычный пароль
- Для Яндекса: https://id.yandex.ru/security/app-passwords

**❌ Form button не кликается**
- Открой F12 → Console
- Проверь что нет красных ошибок JavaScript
- Попробуй перезагрузить страницу (F5)

**❌ Письма не синхронизировались**
- Проверь что в Яндекс почте есть письма
- Может быть нужно подождать 5-10 секунд синхронизации
- Смотри backend логи для ошибок IMAP

## 📊 API примеры (для продвинутых)

### Подключить аккаунт (curl)
```bash
curl -X POST http://localhost:8000/api/email/accounts/connect \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Yandex",
    "provider": "generic",
    "username": "user@yandex.com",
    "password": "16char_app_password",
    "imap_server": "imap.yandex.com",
    "imap_port": 993,
    "smtp_server": "smtp.yandex.com",
    "smtp_port": 465,
    "use_ssl": true
  }'
```

### Получить все аккаунты
```bash
curl http://localhost:8000/api/email/accounts
```

### Получить все треды
```bash
curl http://localhost:8000/api/email/threads
```

### Отправить письмо
```bash
curl -X POST http://localhost:8000/api/email/send \
  -H "Content-Type: application/json" \
  -d '{
    "account_id": 1,
    "to": ["lead@example.com"],
    "subject": "Спецпредложение",
    "body": "Привет!",
    "lead_id": 123
  }'
```

## 📚 Технические детали

### Установленные пакеты для email
- `aiosmtplib==5.1.0` - асинхронный SMTP клиент
- `imap-tools==1.11.1` - синхронизация писем по IMAP
- `beautifulsoup4==4.14.3` - парсинг HTML писем
- `email-validator==2.3.0` - валидация email адресов

### Модели БД
- `EmailAccount` - подключённые почтовые аккаунты
- `EmailThread` - ветки переписки (темы писем)
- `EmailMessage` - отдельные письма
- `EmailCampaign` - кампании рассылок

### API маршруты
Все маршруты начинаются с `/api/email`

## ✨ Что делает система при подключении

1. **Проверяет учётные данные** - пытается подключиться к IMAP
2. **Синхронизирует письма** - загружает последние письма
3. **Анализирует письма** - извлекает email отправителя, тему, текст
4. **Ищет связь с лидом** - пытается найти лида по email отправителя
5. **Создаёт треды** - группирует письма по темам
6. **Сохраняет в БД** - сохраняет всё в PostgreSQL
