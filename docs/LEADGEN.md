# Лидогенерация SmartCRM

Дата: апрель 2026  
Статус: реализовано (MVP)

---

## Что это

AI-лидогенератор: вводишь ИНН, название компании или описание портрета клиента — система автоматически собирает всю доступную информацию, прогоняет через 5 анализов и 4 агента, выдаёт карточку с ЛПР, скором 0–100, стратегией захода и рекомендованными продуктами.

---

## 3 режима работы

### 1. Прямой (ИНН / название)
Знаешь конкретную компанию → вводишь ИНН или название → получаешь полную карточку.

### 2. По портрету
Описываешь портрет идеального клиента текстом или голосом:  
> «IT-компания в Москве, 50–200 сотрудников, нет CRM, активно нанимает»

Hermes парсит портрет → структурированные критерии → список компаний.  
Можно нажать «Анализировать» на любой найденной компании.

### 3. Кластер / Холдинг
Вводишь ИНН якорной компании → система строит карту всей группы:
- Учредители-юрлица и их компании  
- Другие компании общих учредителей-физлиц  
- Суммарный оборот группы

## Архитектура пайплайна

```
Вход (ИНН / название / портрет)
  │
  ▼
[Идентификация] DaData → компания, директор, учредители
  │
  ▼ Параллельно:
  ├── ФНС / bo.nalog.ru → финансы, тренд, арбитраж
  ├── BuiltWith → технологический стек
  ├── NewsAPI + WebSearch → свежие новости (макс 6 мес.)
  └── Buster → email ЛПР (паттерн или API)
  │
  ▼
[5 анализов профиля]
  ├── IT-зрелость → что предлагать
  ├── Структура решений → кому писать (ЛПР)
  ├── Вендор-ландшафт → как позиционировать
  ├── Траектория роста → какой аргумент
  └── Безопасность/комплаенс → какой продукт
  │
  ▼ Параллельно:
  ├── Аналитик → надёжность, BANT, скор
  ├── Тех. специалист → совместимость, продукты
  ├── Маркетолог → триггеры, крючок, срочность
  └── Стратег → план захода, скрипт
  │
  ▼
[Экономист] → итоговый скор (profile + agents)
  │
  ▼
Карточка лида
  ├── ЛПР (имя, должность, email + SMTP-проверка)
  ├── Учредители + связи
  ├── Финансы (выручка, прибыль, тренд)
  ├── Технологии (стек, зрелость, пробелы)
  ├── 5 анализов профиля
  ├── Скоры 4 агентов + итог
  ├── Крючок + скрипт + рекомендованные продукты
  └── Новости + триггеры
```

## Источники данных

| Источник | Что даёт | ENV ключ | Лимит |
|----------|----------|----------|-------|
| **DaData** | ИНН, компания, директор, учредители, ОКВЭД | `DADATA_API_KEY`, `DADATA_SECRET_KEY` | 10 000/сут |
| **ФНС / bo.nalog.ru** | Финансы, тренд выручки | нет (бесплатно) | мягкий |
| **BuiltWith** | Технологический стек | `BUILTWITH_API_KEY` | 1 req/sec |
| **NewsAPI** | Свежие новости (≤ 180 дней) | `NEWS_API_KEY` | 100/день |
| **Brave/Serper/Tavily** | Веб-поиск (уже есть) | уже в `.env` | см. SEARCH.md |
| **Buster** | Email ЛПР | `BUSTER_API_KEY` | платный |

Все источники работают с **fallback**: если API недоступен или нет ключа — шаг пропускается, пайплайн продолжается.

## 5 анализов профиля

### 1. IT-зрелость
**Данные:** BuiltWith (стек), новости (найм IT)  
**Вывод:** `basic` / `optimization` / `enterprise`  
**Применение:** какой продукт предлагать

### 2. Структура решений  
**Данные:** DaData (директор, учредители), ФНС (филиалы)  
**Вывод:** `owner_managed` / `hired_director` / `board`  
**Применение:** кому писать и как обращаться

### 3. Вендор-ландшафт
**Данные:** BuiltWith (текущий стек)  
**Вывод:** `replace` / `integrate` / `complement`  
**Применение:** как позиционировать наш продукт

### 4. Траектория роста
**Данные:** ФНС (динамика выручки), новости (инвестиции, найм)  
**Вывод:** `growing` / `stable` / `declining`  
**Применение:** какой аргумент работает

### 5. Безопасность / комплаенс
**Данные:** ФНС (ОКВЭД, лицензии), новости (инциденты)  
**Вывод:** `basic` / `certified` / `incident_driven`  
**Применение:** ManageEngine vs PosTech vs другой продукт

## Скоринг

```
Итоговый скор = profile_score_impact (макс 40) + agent_avg × 0.6
```

- `profile_score_impact` — сумма вкладов 5 анализов (макс 40 баллов)
- Каждый агент даёт скор 0–100, берётся среднее, умножается на 0.6

| Скор | Приоритет | Действие |
|------|-----------|----------|
| 80–99 | critical | Звони сейчас |
| 60–79 | high | Запланируй звонок |
| 40–59 | medium | Изучи подробнее |
| 0–39 | low | Мониторь |

## Голосовые команды

Все команды работают и голосом и текстом:

```
"найди компанию по ИНН 7736207543"
"сгенерируй лид для ООО Ромашка"
"найди лиды: IT-компании в Москве с 50+ сотрудников"
"найди холдинг этой компании 7736207543"
```

Hermes автоматически определяет интент и направляет на страницу `/leadgen`.

## Ops UI (обучение без разработчика)

На странице `/leadgen` → кнопка **⚙ Настройки**:

| Параметр | Что меняет |
|----------|-----------|
| Мин. скор для CRM | При `save_to_crm=true` лиды ниже порога не сохраняются |
| Макс. возраст новостей | Фильтр NewsAPI (по умолчанию 180 дней) |
| Доп. промпт аналитика | Дописывается к системному промпту аналитика |
| Доп. промпт тех. специалиста | Кастомизация анализа стека |
| Доп. промпт маркетолога | Кастомизация триггеров |
| Примеры портретов | Быстрые кнопки на странице поиска по портрету |

Конфиг сохраняется в `data/leadgen_config.json`.
