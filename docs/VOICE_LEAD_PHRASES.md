# Голосовые формулировки — лиды (эталон для Hermes и UI)

Документ фиксирует, **как пользователь может сказать команду** и что должен вернуть роутер.  
Правки промпта Hermes (`backend/core/hermes.py`) и UI (`leads/+page.svelte`) сверяйте с этим списком.

## Создать лид (`create_lead`)

- «Создай лид компания ООО Ромашка»
- «Добавь в базу ГК Логист, контакт Петр»
- «Новый лид: ООО «Север», телефон плюс семь девятьсот…»
- **Нельзя:** «создай два лида» без названий → лучше `noop` с просьбой уточнить (по одному лиду за команду).

Слоты: `company` (обязательно), `contact`, `phone`, `email`, `budget`, `note`.

## Изменить лид (`update_lead`)

Идентификация лида (одно из):

- `company` — полное или короткое уникальное имя («Ромашка», «ООО Ромашка»)
- `lead_id` — число (`id` в таблице) **или** текстовое имя компании, если модель не выдала `company`
- `number` — номер строки в **текущем отфильтрованном** списке

Формулировки:

- «В лиде «Ромашка» исправь почту на ivan@mail.ru»
- «У ООО Ромашка поменяй телефон на +7 916…»
- «В лиде ромашка почту исправь на …» (модель: `company` или `lead_id: "ромашка"`)

Hermes должен отдавать **`field` + `value`** для смены поля:

| Смысл   | `field` (пример) | `value`      |
|---------|------------------|--------------|
| Почта   | `email`, «почту» | email        |
| Телефон | `phone`, «номер» | телефон      |
| Контакт | `contact`        | ФИО          |
| Этап    | `stage`          | Новый / …    |
| Бюджет  | `budget`         | сумма        |

Прямые слоты `email`, `phone` без `field` тоже поддерживаются UI.

## Удалить лид (`delete_lead`)

- «Удали лид ООО Ромашка»
- «Убери компанию из базы, МашПром»

Избегать только «ООО» без названия — неоднозначно.
