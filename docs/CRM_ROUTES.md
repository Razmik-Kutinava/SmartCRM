# SmartCRM — маршруты CRM и Лиды

Раздел **«Лиды»** в приложении — основной UI для воронки, списка, календаря, задач, фокуса и аналитики. Путь **`/crm/*`** оставлен для **обратной совместимости** и букмарклетов: все адреса **`/crm/...`** отдают редирект **308** на соответствующие страницы **`/leads/...`**. Отдельного дублирующего дерева Svelte-компонентов под `/crm` нет — один код, два префикса URL.

---

## Таблица редиректов

| Запрос пользователя | Куда ведёт редирект (308) |
|---------------------|---------------------------|
| `/crm` | `/leads/list` |
| `/crm/list` | `/leads/list` |
| `/crm/funnel` | `/leads/funnel` |
| `/crm/calendar` | `/leads/calendar` |
| `/crm/tasks` | `/leads/tasks` |
| `/crm/focus` | `/leads/focus` |
| `/crm/analytics` | `/leads/analytics` |
| `/crm/{id}` | `/leads/{id}` |

Реализация: `frontend/src/routes/crm/**/+page.js` (отдельный `load()` с `redirect`).

---

## Поведение в коде

- Таб-навигация по разделам лидов: `frontend/src/routes/leads/+layout.svelte` (подпись: карточка сделки из списка или воронки; **`/crm` ведёт сюда же**).
- Голосовой контекст и навигация по интентам обрабатывают и **`/leads`**, и **`/crm`** как одну зону «лиды» — см. `pageContext()` и `_handleIntentNav` в `frontend/src/routes/+layout.svelte`.
