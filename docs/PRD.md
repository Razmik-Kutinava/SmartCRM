# SmartCRM — Product Requirements Document

## Идея проекта

SmartCRM — это автономная система замены отдела продаж. Один человек управляет 5 специализированными AI-агентами, которые работают параллельно в фоне и выполняют команды через голосовой интерфейс.

**Цель:** Один оператор = полноценный отдел продаж.

---

## Проблема

Отдел продаж — дорого, медленно, человеческий фактор. Рутина (поиск лидов, анализ, письма, отчёты) занимает 80% времени и не требует творческого мышления. AI может делать это лучше, быстрее и 24/7.

---

## Решение

Мультиагентная CRM-система с:
- Голосовым управлением (говоришь — агенты исполняют)
- 5 специализированных агентов уровня PhD в своей области
- Автономным поиском информации (Brave, Tavily, Serper)
- RAG-базой знаний которая пополняется сама
- **Groq** (основной для агентов и часто для Hermes) + **Ollama / Qwen** локально (fallback агентов, Hermes по политике `HERMES_ROUTING_POLICY`)

---

## Агенты

| Агент | Роль |
|-------|------|
| Аналитик | Анализ данных лидов, отчёты, прогнозы, метрики |
| Стратег | Слушает всех агентов, принимает решения, даёт советы |
| Экономист | Финансовый анализ, расчёты, экономические прогнозы |
| Маркетолог | Исследует клиента, персональные письма, 20 касаний |
| Тех. специалист | Экспертиза продукта, технические знания |

---

## MVP (Фаза 1)

- [ ] Голосовые команды → управление лидами (CRUD)
- [ ] Веб-интерфейс (SvelteKit) на русском языке
- [ ] Groq Whisper → Hermes роутер → агент → действие
- [ ] База лидов как в Битрикс (полный набор полей)
- [ ] 1 пользователь, локально

## Фаза 2

- [ ] Поиск через Brave + Tavily + Serper → RAG
- [ ] Персональные письма и цепочки касаний (маркетолог)
- [ ] Аналитика и отчёты (экономист + аналитик)
- [ ] Параллельная работа всех 5 агентов

## Фаза 3

- [ ] Telegram бот (полноценный)
- [ ] Экспорт в Excel/PDF
- [ ] Автономный поиск новых лидов агентами
- [ ] Интеграция с внешними CRM

## Стек

```
Frontend:  SvelteKit + Tailwind + Vite
Backend:   FastAPI + asyncio + PostgreSQL + Redis
Агенты:    LangGraph (параллельные)
LLM:       Groq (агенты) → Ollama (fallback); Hermes: Groq и/или Ollama — см. docs/stack/LLM.md
Роутер:    Hermes — парсинг интентов (JSON)
Голос:     Groq Whisper → текст → Hermes
RAG:       Brave/Tavily/Serper → rag/retrieve + Chroma → агенты
Инфра:     Docker + docker-compose + Nginx
```

---

## Принципы

1. **Голос — основной интерфейс**, кнопки — вспомогательные
2. **Агенты работают параллельно**, стратег координирует
3. **Groq → Ollama fallback** — не зависим от одного провайдера
4. **RAG пополняется автоматически** — система умнеет со временем
5. **Язык интерфейса — русский**
6. **Приватность** — Ollama остаётся локально; запросы к Groq уходят в облако там, где это допустимо по политике (см. `docs/stack/LLM.md`, `docs/RUNBOOK.md`)
