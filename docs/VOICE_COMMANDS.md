# SmartCRM — Голосовые команды

## Обзор

Голосовой интерфейс SmartCRM доступен **на любой странице** приложения.  
Панель расположена в верхней части экрана (под сайдбаром), работает через WebSocket.

### Архитектура пайплайна

```
Микрофон / Текст
       ↓
  WebSocket /ws/voice
       ↓
  voice/pipeline.py
       ↓
  Groq Whisper (STT)  ← только для аудио
       ↓
  core/hermes.py  ←  Hermes Router (Groq/Ollama)
       ↓         [page_context передаётся автоматически]
  intent JSON:
  { intent, agents, slots, reply }
       ↓
  agents/orchestrator.py
  ┌─────────────────────────────────┐
  │  ask_economist → economist      │
  │  ask_marketer  → marketer       │  + поиск в интернете
  │  ask_tech      → tech_specialist│
  │  ask_strategist→ strategist     │
  │  search_web    → поиск (free)   │
  │  write_email   → marketer       │
  │  run_analysis  → все агенты     │  параллельно
  │  create_lead   → analyst        │
  │  update_lead   → analyst+...    │
  │  list_leads    → analyst        │
  └─────────────────────────────────┘
       ↓
  final_reply → WebSocket → UI
```

## Панель голосового ввода

### Элементы

| Элемент | Описание |
|---------|----------|
| 🎙 Кнопка | Нажать — начать запись. Нажать снова — остановить и отправить. |
| Текстовое поле | Написать команду, нажать Enter — то же что голос. |
| Метка страницы | Показывает текущую страницу — передаётся в Hermes как контекст. |
| Индикатор WS | Зелёный/красный — статус WebSocket соединения. |
| 👍👎 | Обратная связь — правильно ли был распознан интент. |

### Контекст страницы

Hermes автоматически получает информацию о текущей странице:
- `Страница: Лиды` — подсказывает, что команды "добавь заметку", "обнови" относятся к лидам
- `Страница: Поиск` — помогает понять, что нужен поиск
- `Страница: Дашборд` и т.д.

## Голосовой пайплайн — технические детали

### Файлы

| Файл | Роль |
|------|------|
| `frontend/src/routes/+layout.svelte` | Глобальная панель голосового ввода |
| `frontend/src/lib/websocket.js` | WS-клиент, `sendText(text, pageContext)` |
| `backend/api/routes/voice.py` | WS endpoint `/ws/voice`, REST `/api/voice/command` |
| `backend/voice/pipeline.py` | `process_text(text, page_context)`, `process_audio()` |
| `backend/core/hermes.py` | Intent router, системный промпт с примерами |
| `backend/agents/orchestrator.py` | Маршрутизация по интенту, `_run_voice_query()` |

### Поток данных (text command)

```
layout.svelte (voiceSubmitText)
  → sendText(text, pageContext)       # websocket.js
  → WS: {type:"text", text, page_context}
  → voice.py WebSocket handler
  → pipeline.process_text(text, page_context)
    → hermes_text = f"{page_context}\n{text}"
    → parse_intent(hermes_text)       # Hermes
    → run_agents(intent, slots)       # Orchestrator
      → _run_voice_query()            # для новых интентов
        → free_search(question)       # Serper+Brave+Tavily
        → core.llm.chat(prompt)       # Groq/Ollama
      → final_reply
  → WS: {type:"intent", intent, reply, agents, trace_id}
  → layout.svelte: показывает ответ
```

### Как агент использует поиск

Для `ask_*` интентов оркестратор:
1. Берёт `question` и `company` из слотов
2. Вызывает `free_search(f"{company} {question}", max_results=8)`
3. Добавляет результаты в промпт агента
4. Агент (через `core.llm.chat`) формирует итоговый ответ с данными из поиска

## Настройка и расширение

### Добавить новый интент

1. **Hermes** (`core/hermes.py`): добавить интент в список и примеры в `SYSTEM_PROMPT`
2. **Orchestrator** (`agents/orchestrator.py`):
   - добавить в `_AGENT_INTENTS`
   - добавить в `_INTENT_TO_AGENTS` если нужен особый агент
   - при необходимости добавить ветку в `_run_voice_query()`

### Пример добавления интента `ask_lawyer`

```python
# В _AGENT_INTENTS:
_AGENT_INTENTS = frozenset({
    ...,
    "ask_lawyer",
})

# В _INTENT_TO_AGENTS:
_INTENT_TO_AGENTS = {
    ...,
    "ask_lawyer": ["analyst"],  # или новый агент
}
```

## Примеры реальных сценариев

### Сценарий 1: Финансовый анализ конкурента
```
Пользователь (на странице Поиск): 
"какая выручка у Яндекса за 2025 год"

→ Hermes: ask_economist
→ Orchestrator: _run_voice_query (economist)
  → free_search("Яндекс какая выручка за 2025 год")
  → Groq: анализ данных из поиска
→ Ответ: "Выручка Яндекса за 2025 год по данным отчётности составила..."
```

### Сценарий 2: Создание лида с контекстом
```
Пользователь (на странице Лиды):
"добавь компанию ТехноПром из Казани, бюджет 2 миллиона"

→ Hermes (контекст: Страница: Лиды): create_lead
→ Orchestrator: analyst
→ Создан лид: ТехноПром, город Казань, бюджет 2000000
→ Страница лидов обновляется
```

### Сценарий 3: Полный анализ клиента
```
Пользователь (любая страница):
"проанализируй лид Альфа Строй"

→ Hermes: run_analysis
→ Orchestrator: параллельно analyst + economist + marketer + tech_specialist
→ Стратег итожит всё в final_reply
→ Ответ: комплексный анализ со скором, финансами, конкурентами, стеком
```

### Сценарий 4: Написание письма
```
Пользователь: "напиши холодное письмо компании Вектор про внедрение CRM"

→ Hermes: write_email {target: "Вектор", topic: "внедрение CRM", tone: "formal"}
→ Orchestrator: _run_write_email (marketer)
  → free_search("Вектор внедрение CRM")  — обогащение контекстом
  → Groq: генерация письма
→ Ответ: готовый текст письма
```
