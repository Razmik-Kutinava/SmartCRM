## Технические замечания

**Проблема Groq Rate Limit:**  
При параллельном запуске тестов без паузы Groq достигает лимита 6000 TPM для `llama-3.1-8b-instant`, переключается на Ollama (офлайн) и тесты падают. Решение: пауза 5 секунд между тестами в скрипте.

**Ограничение текущей модели:**  
`llama-3.1-8b-instant` — небольшая модель. При богатых входных данных (много полей + chain-of-thought) работает корректно. При rate limit деградирует до Ollama fallback.

## Что тестируется в скрипте

**Файл:** `backend/tests/test_agents.py`

| Проверка | Смысл |
|---|---|
| score в диапазоне | Агент не выдаёт фантастические числа |
| bant-поля заполнены | Структурированный ответ есть |
| НЕ выдумывает ЛПР | Нет галлюцинации при пустых данных |
| Имя клиента в письме | Реальная персонализация |
| Стек соответствует отрасли | Знание предметной области |
| LTV/бюджет при неизвестном | Умеет оценивать по косвенным |
